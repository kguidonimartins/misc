# misc — agent instructions

Personal R package for data wrangling, Excel I/O, and geospatial helpers (`sf`, GDAL).
Repo: https://github.com/kguidonimartins/misc

## Commands

Prefer the `Makefile` targets when they exist:

```bash
make install_deps   # remotes::install_deps()
make document       # devtools::document()
make tests          # devtools::test() → out-testthat.txt
make check          # devtools::check() → out-check.txt
make check-cran     # CRAN-like check → out-check-cran.txt
make styler         # styler::style_dir('R')
make eg             # devtools::run_examples() → out-eg.txt
make extdata        # regenerate inst/extdata samples
```

Equivalent one-liners:

```bash
Rscript -e "devtools::load_all()"
Rscript -e "devtools::test()"
Rscript -e "devtools::test(filter = '^read_geo')"
Rscript -e "devtools::document()"
Rscript -e "devtools::check()"
```

During interactive work, use `devtools::load_all()` (not `library(misc)`) so unexported helpers are available.

## Code style

- Match existing files in the same area before introducing new patterns.
- Use the **magrittr pipe** (`%>%`). This package imports `magrittr`; do not switch to `|>` unless the maintainer asks for a repo-wide migration.
- Use `\(...) ...` for single-line anonymous functions; use `function() { ... }` otherwise.
- Internal helpers: dot-prefix and **no** roxygen (e.g. `.read_geo_check_path()`).
- After editing `R/`, run `make styler` (this repo uses **styler**, not `air`). Note: `styler` is a Suggests dep; install it manually if missing.
- Soft dependency on packages like `magick` is handled via `check_require("magick")` from `R/utils.R` — do not add the package to Imports just for one function.
- Geospatial code: prefer `sf` idioms, `fs` for paths, and explicit CRS handling. Many readers return structured lists with metadata plus a `data` column.
- RStudio project uses 2-space indentation.

## File layout

| Code | Tests | Notes |
|------|-------|-------|
| `R/{name}.R` | `tests/testthat/test-{name}.R` | Mirror structure |
| `inst/extdata/` | accessed via `system.file("extdata", ..., package = "misc")` | Spatial fixtures |
| `inst/xlsx-examples/` | accessed via `system.file("xlsx-examples", ..., package = "misc")` | Excel workbooks for read-all-sheets tests |
| `data-raw/` | — | Scripts to build extdata; not run in checks |
| `README.Rmd` | — | **Edit README.Rmd**, not `README.md` (generated) |

`R/` files without test coverage: `deduplicate_by.R`, `remove_columns_based_on_NA.R`, `view_in.R` (viewers are macOS-only, hard to test).

## Testing (testthat)

Tests live in `tests/testthat/`. Hierarchy: **file → `test_that()` → expectation**.

```r
test_that("descriptive behavior", {
  result <- my_function(input)
  expect_equal(result, expected)
})
```

Runners:

```r
devtools::test()                                    # full suite
devtools::test(filter = "read_geo")                 # by pattern
testthat::test_file("tests/testthat/test-foofy.R")  # single file
```

Existing tests use `on.exit()` for cleanup and `skip_if_not_installed()` / `skip_if_not()` / `skip_on_cran()` for conditional skipping. For new tests, prefer `withr::local_tempfile()` / `local_tempdir()` patterns for cleaner cleanup.

Avoid deprecated testthat 2 patterns: `context()`, `with_mock()`, `expect_equivalent()` (use `expect_equal(ignore_attr = TRUE)`).

## Documentation

- Every **exported** function needs roxygen2 with `@export` and `@examples` where sensible.
- Use `@family` tags consistently (e.g. `missing-data`, `geo-io`, `data-viewers`).
- Wrap roxygen comments at **80 characters**.
- Do **not** document internal (non-exported) functions.
- After roxygen changes: `make document`.
- Add new exported topics to `_pkgdown.yml` reference index.

## NEWS.md

- User-facing changes get a bullet; skip tiny doc-only or internal refactors.
- Mention the function name early in the bullet.
- Reference GitHub issues/PRs as `(#123).` before the final period.
- Order bullets alphabetically by function name; general bullets without function names go first.

## CI

GitHub Actions (`R-CMD-check.yaml`): 5 jobs — macOS (release), Windows (release), Ubuntu (devel, release, oldrel-1). `pkgdown.yaml` builds and deploys site to `gh-pages` branch. `rhub.yaml` for manual R-hub dispatch.

## Domain notes

- **View helpers** (`view_vd`, `view_vd_nonint`, `view_mapview_from_path`) are **macOS-only**; preserve platform checks and clear errors on other OSes.
- **Extdata** — spatial samples in `inst/extdata/`; regenerate with `make extdata` after changing fixture builders in `data-raw/`.
- **Excel fixtures** — workbooks in `inst/xlsx-examples/` generated by `inst/xlsx-examples.R`; regenerate manually when test expectations change.
- **GDB** — `read_gdb()` tests skip automatically when the GDAL `OpenFileGDB` driver cannot write; this is expected on some platforms.
- **`.env`** — contains a `GITHUB_PAT`; listed in `.gitignore` and `.Rbuildignore`. Never commit its contents.
- **`check_require()`** — internal helper in `R/utils.R` used to check for suggested packages before using them (e.g., `magick` in `trim_fig`).

## What agents should not do

- Edit generated `README.md` directly.
- Commit secrets from `.env` or commit changes to `.env`.
- Write test output or scratch files into `R/` or the package root.
- Run `devtools::release()` or `submit_cran` without explicit maintainer request.
- Force-push or amend published commits unless asked.
- Add a package to `Imports` when it could be a soft dependency (use `check_require()` and add to `Suggests` instead).
